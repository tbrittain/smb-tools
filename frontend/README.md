# Frontend (Vue 3)

Vue 3 + TypeScript frontend for smb-tools, bundled by Vite and embedded in the desktop app via [Wails v2](https://wails.io/).

## Setup

Requires Node.js 26

```sh
npm install
```

## Commands

```sh
npm run dev        # Vite dev server (used by Wails in dev mode)
npm run build      # TypeScript check + production build
npm run lint       # Biome lint check
npm run lint:fix   # Biome auto-fix
npm run test       # Vitest unit tests (watch mode)
npm run test:run   # Vitest unit tests (one-shot)
npm run storybook  # Storybook component explorer on :6006
```

## Standalone mode

The frontend can run without the Go backend for pure UI work:

```sh
npm run dev
```

Wails bindings are not available in standalone mode. Mock or stub any backend calls when developing UI in isolation. [Storybook](https://storybook.js.org/) is the preferred environment for component development without a live save game.

## Wails bindings

Backend methods are imported from `../../wailsjs/go/main/App` and called as async functions. The files under `wailsjs/go/` are autogenerated. Run `wails build` from the repo root after any Go binding changes to regenerate them.

## Stack

- [Vue 3](https://vuejs.org/) with `<script setup>` Composition API
- [TypeScript](https://www.typescriptlang.org/) strict mode
- [Vite](https://vitejs.dev/)
- [PrimeVue 4](https://primevue.org/) - component library
- [Pinia](https://pinia.vuejs.org/) — global state
- [vue-router](https://router.vuejs.org/) — hash history (required for Wails)
- [Apache ECharts](https://echarts.apache.org/) + [vue-echarts](https://github.com/ecomfe/vue-echarts) — charts
- [Biome](https://biomejs.dev/) — linting and formatting
- [Vitest](https://vitest.dev/) — unit tests
- [Storybook](https://storybook.js.org/) — component development and visual testing
